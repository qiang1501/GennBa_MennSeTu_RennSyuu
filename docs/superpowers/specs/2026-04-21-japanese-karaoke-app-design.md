# 日本語発声カラオケ採点システム（MVP）設計書

## プロジェクト概要
React, TypeScript, Viteを利用し、入力された日本語の文章をカラオケ形式で表示し、読み上げた音声をWeb Speech APIで認識・採点するWebアプリケーション。

## 1. システムアーキテクチャ

* **フロントエンドフレームワーク**: React 18, TypeScript, Vite
* **形態素解析**: `kuromoji.js` (ブラウザ内で動作。辞書は静的ファイルとしてロード)
* **音声認識**: Web Speech API (`webkitSpeechRecognition` / `SpeechRecognition`)
* **スタイリング**: Vanilla CSS（レスポンシブ対応）

## 2. コア機能要件

### テキスト分割と解析
1. **行分割**: 入力テキストを改行で分割する
2. **単語分割**: 各行を `kuromoji` で単語単位に分割する
3. **データ構造**: 各単語に `id`, `surface` (表示用), `reading` (採点用カタカナ), `status` (pending | correct | incorrect) を持たせる

### 音声認識とライブ表示
* `useSpeechRecognition` カスタムフックにて Web Speech API をラップ。
* `continuous = true`, `interimResults = true`, `lang = "ja-JP"` を設定。
* 暫定結果 (`interim`) は画面の**ライブ表示用**（現在発声中の内容を可視化）のみに使用し、採点には絶対に使わない。

### 確定採点アルゴリズム（完全一致 × 連続発話対応）
* **正規化処理**: 
  * 「ひらがな → カタカナ」の変換
  * 記号や空白の除去
  * 全角/半角の統一
* **採点方法（アプローチX）**:
  1. 音声認識から確定結果 (`isFinal = true`) が返ってきた際、その塊（例：「キオンガ」）をさらに `kuromoji` に通して単語分割し、「認識用単語配列」を作成する。
  2. 認識用単語配列の各単語と、現在の「お題の単語 (`reading`)」を**完全一致 (`===`)** で比較する。
  3. 部分一致、意味的な推測、文脈補完は一切行わない。
  4. 一致すれば `correct`（青）、不一致なら `incorrect`（赤）とし、どちらに転んでも次の単語へ判定対象を進める。

## 3. UI/UX 仕様

1. **入力エリア＆コントロール**: テキスト入力欄、分析開始ボタン、録音開始・停止・リセットボタン
2. **カラオケ表示エリア**:
   * 行ごとに単語を横並びで表示
   * 未判定：グレー
   * 現在の対象単語：ハイライト（枠線や背景色で強調）
   * 正解：青
   * 不正解：赤
3. **ライブ進捗**: 音声認識の `interim` 結果を別枠でリアルタイム表示
4. **スコアパネル**: 全行完了後、総単語数・正解数・不正解数・正答率・満点換算（100点）を表示

## 4. コンポーネント構成（src/components）

* `TextInputPanel.tsx`: 課題テキストの入力と分析開始操作
* `KaraokeLine.tsx`: 単語単位のハイライト/正誤色分け表示
* `Controls.tsx`: 録音の開始・停止等の操作
* `LivePreview.tsx`: 暫定認識結果（interim）の表示
* `ScorePanel.tsx`: すべて終了後の結果表示画面

## 5. エラーハンドリング・制約事項

* テキスト未入力時の分析防止
* 音声認識未対応ブラウザでのフォールバックメッセージ
* マイク権限設定の拒否時のメッセージ
* 認識が止まった場合の復旧・再開の手段は今後の拡張とする（MVPではエラー通知のみ対応）
